### 수정 내용

- mock off 상태에서는 화면이 API 응답을 받기 전까지 빈 상태 또는 0 값으로 시작하도록 정리했다.

- 로그인/회원가입 입력 기본값도 mock 모드에서만 데모 계정이 보이도록 변경했다.

### 검증

- Vite 빌드에서 500KB 이상 chunk 경고가 표시되었지만, 빌드는 성공했다.

### 남은 개선점

- 실제 백엔드 API가 연결되면 각 화면의 빈 상태 문구를 API 오류, 데이터 없음, 권한 없음으로 더 세분화할 수 있다.
- 분석 실행 ID를 전역 상태 또는 URL에 보관하면 `latest` 의존을 줄일 수 있다.
