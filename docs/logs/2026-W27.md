  - 로그인 후: 홈, 프로젝트, 새 분석, 진행, 결과, 상세, 커밋 메시지, 분석 내역, 설정 접근
- 로그인 유지 UI 흐름 보강
  - 앱 시작 시 `/auth/refresh` 호출 기준으로 세션 복구
  - 로그인 상태에서 주기적으로 세션 갱신 시도
  - 사이드바 하단 로그아웃 버튼을 명확히 노출
- 주요 페이지 1차 복구
  - 랜딩, 로그인, 회원가입
  - 로그인 후 홈 대시보드
  - 프로젝트 목록
  - 새 분석 시작
  - 분석 진행 상태
  - 분석 결과 요약
  - 파일 상세 분석
  - 커밋 메시지 생성
  - 분석 내역
  - 마이페이지/설정
- 분석 방향성 반영
  - Git 명령어 산출물 업로드 분석
  - GitHub 저장소 연동 분석
- 차트 반영
  - `recharts` 기반 활동 라인 차트, 변경 유형 도넛 차트, 메시지 타입 바 차트 추가
  - 차트 클릭 시 포커스 라인이 남는 문제를 CSS에서 방지
- 사이드바 반응형 보정
  - 노트북처럼 높이가 낮은 화면에서 메뉴 간격과 프로필 영역을 자동으로 축소
  - 사이드바 내부 메뉴 영역만 스크롤되도록 조정해 로그아웃 버튼이 잘리지 않게 개선
- 주요 페이지 UI/UX 재정리
  - `projects`: 검색/필터/정렬/보기 전환 영역과 프로젝트 카드 정보 구조 개선
  - `analysis/new`: 분석 방식, 기본 정보, 옵션, 업로드 명령어 가이드를 단계형 화면으로 재구성
  - `analysis/detail`: 파일 요약, 코드/AI/커밋 탭, 리뷰 포인트 패널을 읽기 쉽게 재배치
  - `analysis/commit-message`: 추천 메시지 카드와 우측 분석 요약 패널을 분리
  - `settings`: 프로필, 연동 상태, 알림, API 설정 영역을 실제 설정 화면처럼 재구성
- 커스텀 알림 구조 반영
  - 확인/경고: SweetAlert2
  - 성공/실패/로딩: Sonner
  - 긴 설정창: Radix Dialog
- 더미 데이터 모드 반영
  - `VITE_USE_MOCKS=true|false`로 더미 데이터 표시 여부 전환
- 백엔드 참고 반영
  - `.env` 값을 `settings.py`에서 읽는 방식으로 조정
  - MariaDB를 실제 인증/서비스 DB 기준으로 정리
  - PostgreSQL(pgdb)은 보조 컨테이너로 유지

## 검증 내용

- 백엔드 import 확인: `uv run python -c "import main; print(main.app.title)"`
- 백엔드 compile 확인: `python -m compileall main.py core routes schemas utils`
- MariaDB 실제 테이블 확인: `users`, `projects`, `analysis_runs`, `uploaded_files`, `changed_files`, `risk_findings`, `ai_outputs`
- 프론트 깨진 문자열 패턴 검색 통과
